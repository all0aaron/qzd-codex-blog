PHASE 0 需求分析

提出日期：2026-06-15。目标是开发一个有前台和后台的博客系统网站。
已完成

项目定位

系统用于发布、管理和浏览长文章内容，重点服务个人博客、知识沉淀、项目日志、技术文章和专题内容运营。前台强调阅读体验、检索效率和内容发现；后台强调文章生产、发布控制、评论治理和站点配置。

角色拆解

角色核心诉求主要权限
游客浏览文章、搜索内容、查看分类标签和文章详情。访问公开内容，提交评论需填写昵称和邮箱。
管理员高效写作、发布文章、管理评论、维护站点信息。文章、分类、标签、评论、友链、站点设置全量管理。
编辑撰写草稿、维护内容质量。创建和编辑文章，提交发布，查看评论。

功能模块

前台模块

  • 首页：精选文章、最新文章、热门标签、作者信息、订阅入口。
  • 文章列表：按分类、标签、关键字、发布时间筛选。
  • 文章详情：正文阅读、目录、相关文章、评论区、上一篇/下一篇。
  • 归档页：按年月聚合文章，便于查找历史内容。
  • 关于页：作者介绍、联系方式、站点说明。

后台模块

  • 登录鉴权：管理员登录、退出、会话保护。
  • 仪表盘：文章数、阅读量、评论状态、近期内容概览。
  • 文章管理：列表、搜索、筛选、新建、编辑、发布、下架、删除。
  • 分类标签：维护分类、标签和内容归属。
  • 评论管理：审核、隐藏、删除、回复。
  • 站点设置：站点标题、简介、作者资料、SEO 信息。

技术方向

博客系统首版适合采用轻量 Web 架构：前端使用原生 HTML/CSS/JavaScript 或 Vue 轻量实现，后台 API 使用 Node.js 或 Java Spring Boot，数据库固定使用 MySQL。由于序章要求后端端口固定为 8080，后续 PHASE 4 将以 8080 提供 API 服务。

当前阶段只做 UI 原型，不提前写正式业务代码。确认 UI 后，建议 PHASE 3/4 使用 Node.js 原生或 Express 风格 API，便于快速落地；数据库阶段输出 `blog_db` 的 MySQL 建表脚本。

PHASE 1 产品设计

完整产品需求文档，后续开发以此为准。
已完成

产品目标

  • 让读者在首页 5 秒内理解站点主题，并能快速进入感兴趣的文章。
  • 让管理员能在后台完成从写草稿到发布、从评论审核到站点设置的闭环。
  • 首版优先保证内容发布、内容浏览、评论管理、基础统计可用。

核心业务流程

流程步骤成功标准
内容发布登录后台 → 新建文章 → 填标题/摘要/分类/标签/正文 → 保存草稿 → 发布前台首页和列表能看到已发布文章。
内容检索进入前台 → 输入关键词或点击分类标签 → 查看列表 → 打开详情筛选结果准确，空结果有清晰提示。
评论治理读者提交评论 → 后台查看待审 → 审核通过或隐藏删除未审核评论不直接公开，后台可追踪状态。
站点维护后台设置标题、简介、作者资料、SEO 关键字前台展示同步更新。

页面清单

页面文件说明
前台首页ui_home.html博客首页，承载精选内容、文章列表和侧边栏。
文章详情ui_post_detail.html正文阅读页，包含目录、评论、相关文章。
后台登录ui_admin_login.html管理员登录入口。
后台仪表盘ui_admin_dashboard.html运营概览、统计数据、近期动态。
文章管理ui_admin_posts.html文章列表、搜索筛选、批量操作。
文章编辑ui_admin_editor.html文章创建和编辑表单。
评论管理ui_admin_comments.html评论审核、隐藏、删除。
站点设置ui_admin_settings.html基础信息、作者资料、SEO 设置。

非功能需求

  • 前台阅读页在移动端不乱版，正文宽度控制在舒适阅读范围。
  • 后台操作按钮有明确状态，危险操作需二次确认。
  • 接口返回格式统一为 { code, msg, data }。
  • 数据库必须使用 MySQL，库名规划为 blog_db。
  • 后端服务端口固定为 8080。
